AgentsPay

Payment infrastructure for AI agents. Give your agents wallets with spending limits, virtual cards, and real-time transaction tracking. No UI — pure API, SDK, CLI, and MCP.

Your agent gets a virtual Visa card linked to its wallet. It can pay anywhere — DoorDash, Amazon, any website. You set the limits once, and the server auto-approves or declines in real-time.

How It Works

1. You deposit money (Stripe Checkout) ↓ 2. Funds land in your platform balance ↓ 3. You fund an agent wallet (with spending limits) ↓ 4. Wallet gets a virtual Visa card ↓ 5. Agent uses the card to pay online ↓ 6. Stripe asks our server: "approve this charge?" ↓ 7. We check balance + limits → Approve ✓ or Decline ✗ ↓ 8. Wallet balance decremented atomically

Key Concepts

Agents

An agent is an AI system that can hold wallets and make payments. Each agent gets a unique account number (AGT-XXXXXXXX) and an API key for authentication.

Wallets

Wallets are budgets with guardrails. Each wallet belongs to one agent and has:

  • Balance — available funds in USD
  • Daily limit — max spend per day
  • Monthly limit — max spend per month
  • Per-transaction limit — max single payment
  • Allowed merchants — optional whitelist
  • Expiry date — optional time-limited access

Virtual Cards

Each wallet can have one virtual Visa card. The card is issued via Stripe Issuing and can be used for online purchases anywhere Visa is accepted. The agent retrieves the card number, CVC, and expiry to fill in checkout forms.

  • Cards inherit wallet spending limits
  • Freezing a wallet auto-freezes its card
  • Closing a wallet cancels its card
  • Authorization decisions happen in <2 seconds

Transactions

Every payment, transfer, and card charge is recorded as a transaction with full metadata, timestamps, and idempotency keys.

Types: PAYMENT (merchant), TRANSFER (agent-to-agent), FUND, WITHDRAW, REFUND, CARD_PAYMENT (virtual card)

Quick Start

1. Register & Login

npm install -g @usectl/cli

agentspay auth register --email you@example.com --password s3cret
agentspay auth login    --email you@example.com --password s3cret

2. Deposit Funds

# Opens Stripe Checkout in your browser
agentspay funding deposit --amount 100

3. Create an Agent

agentspay agent create --name "shopping-agent" --description "Buys supplies"
# Returns: agent_id, account_number (AGT-...), api_key

4. Create & Fund a Wallet

agentspay wallet create \
  --agent-id <AGENT_ID> \
  --name "shopping-budget" \
  --funding 50 \
  --daily-limit 20 \
  --limit-per-tx 10

agentspay wallet fund <WALLET_ID> --amount 25

5. Issue a Virtual Card

agentspay wallet create-card <WALLET_ID>
agentspay wallet card-info   <WALLET_ID>

6. Agent Pays Online

The agent uses its API key to retrieve card details, then fills in checkout forms:

import { AgentsPay } from '@usectl/sdk';

const ap = new AgentsPay({ agentKey: 'agk_...' });

// Get virtual card details
const card = await ap.cards.getDetails('wallet_id');
// card.number, card.cvc, card.exp_month, card.exp_year

// Agent uses these to pay on any website

SDK

npm install @usectl/sdk

Agent Operations (agentKey auth)

import { AgentsPay } from '@usectl/sdk';
const ap = new AgentsPay({ agentKey: 'agk_...' });

// Check balance
await ap.wallets.getBalance();
await ap.wallets.getBalance('specific-wallet-id');

// List wallets
await ap.wallets.list();

// Pay a registered merchant
await ap.pay({
  merchantId: 'merchant-id',
  amount: 9.99,
  description: 'API tokens',
});

// Transfer to another agent
await ap.transfer({
  toAgent: 'AGT-12345678',
  amount: 5.00,
  reason: 'Payment for subtask',
});

// Get virtual card details
await ap.cards.getDetails('wallet-id');

// View transactions
await ap.transactions.list({ limit: 10, type: 'PAYMENT' });

// Look up another agent
await ap.agents.lookup('AGT-12345678');

// Browse merchants
await ap.merchants.list({ search: 'openai' });

// Request a new wallet from owner
await ap.wallets.request({
  name: 'research-budget',
  amount: 100,
  reason: 'Need tokens for research',
});

Admin Operations (masterKey auth)

const ap = new AgentsPay({ masterKey: 'mk_...' });

// Agent management
await ap.admin.agents.create({ name: 'my-agent' });
await ap.admin.agents.list();
await ap.admin.agents.get('agent-id');
await ap.admin.agents.revoke('agent-id');
await ap.admin.agents.rotateKey('agent-id');

// Wallet management
await ap.admin.wallets.create({ agentId: '...', name: 'budget', fundingAmount: 100 });
await ap.admin.wallets.list();
await ap.admin.wallets.fund('wallet-id', 50);
await ap.admin.wallets.freeze('wallet-id');
await ap.admin.wallets.unfreeze('wallet-id');
await ap.admin.wallets.approve('wallet-id');
await ap.admin.wallets.reject('wallet-id');
await ap.admin.wallets.close('wallet-id');

// Card management
await ap.admin.wallets.createCard('wallet-id');
await ap.admin.wallets.getCard('wallet-id');
await ap.admin.wallets.freezeCard('wallet-id');
await ap.admin.wallets.unfreezeCard('wallet-id');

// Funding
await ap.admin.funding.getBalance();
await ap.admin.funding.deposit(100);

// Transactions
await ap.admin.transactions.list({ limit: 50 });

// Webhooks
await ap.admin.webhooks.create('https://example.com/hook', ['payment.completed']);
await ap.admin.webhooks.list();
await ap.admin.webhooks.delete('webhook-id');

MCP Server (for Claude, GPT, etc.)

Let your AI agent manage money autonomously via Model Context Protocol.

npm install -g @usectl/mcp

Add to your MCP config (claude_desktop_config.json):

{
  "mcpServers": {
    "agentspay": {
      "command": "npx",
      "args": ["-y", "@usectl/mcp"],
      "env": {
        "AGENTSPAY_API_KEY": "agk_your_agent_key"
      }
    }
  }
}

Security Model

  • Master Key (mk_...) — full admin access. Create agents, manage wallets, view all transactions.
  • Agent Key (agk_...) — scoped to one agent. Can only access its own wallets and make payments within limits.
  • Agents cannot exceed spending limits, even with valid keys
  • All transactions are idempotent (safe to retry)
  • Card authorizations use row-level locking to prevent double-spend
  • Wallet balance is deducted atomically on card authorization

Architecture

  • API: Express + Prisma + PostgreSQL
  • Payments: Stripe (Checkout for deposits, Issuing for virtual cards)
  • Auth: JWT tokens, bcrypt password hashing
  • Rate limiting: Per-IP and per-key
  • Webhooks: HMAC-signed delivery with retry
